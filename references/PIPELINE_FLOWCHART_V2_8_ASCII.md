================================================================================
                          动态模型策略（新增）
================================================================================

Step 1.5G (Spec-Kit 落地):
  - L1/L2 任务: sonnet (速度优先，降本)
  - L3 任务: opus (质量优先)

Step 4 (修复循环):
  - Round 1-2: sonnet (快速迭代)
  - Round 3: opus (最后一搏，需要创新)

Step 5.5 (回滚决策):
  - Brainstorming 始终 opus (关键决策，不能因降本而降低质量)

预期降本: 35-45% (相比全部使用 opus)

================================================================================
                              独立性规则
================================================================================

原则: 谁参与主要决策，谁就不做最终仲裁

- OpenAI 制定宪法 --> Claude 仲裁计划分歧
- Claude 主方案 --> OpenAI 仲裁实施分歧
- Gemini 始终不做最终仲裁，只做证据和反方意见

================================================================================
                              通知规范
================================================================================

Agent 自推规则:
  - 有消息能力的 agent 应自己发送开始/关键进度/完成/失败消息到职能群

Main 兜底规则:
  - 负责监控群可见性
  - 补发缺失通知
  - 最终交付通知
  - 告警通知

星链默认规则:
  - Step 1.5/3/4/5.5 的执行 agent 默认同时发送到职能群+监控群

================================================================================
                              完成判定
================================================================================

完成信号必须同时满足:
  [✓] 正式文件已写入对应 agent 自己的目录
  [✓] 返回结构化回执 (不能只写 done)
  [✓] 关键阶段通知已发出或由 main 兜底补发

[!] 缺一项都不算真正完成

================================================================================
                            Spawn 重试机制
================================================================================

任何 agent spawn 失败时:
  [1] 第一次失败 --> 立即重试 (相同参数)
  [2] 第二次失败 --> 等待 10 秒后重试
  [3] 第三次仍失败 --> 推送告警到监控群 + 通知晨星 --> 标记 BLOCKED

================================================================================
                              预期收益
================================================================================

- 效率提升: 30-40% (减少 main 等待时间)
- 降本: 35-45% (优化模型使用 + Brainstorming 动态分配)
- 质量提升: 宪法基于历史经验，避免重复踩坑
- 成功率: Epoch 成功率提升 15-20%
- 用户体验: 主私聊不再"像不可用"

================================================================================
                              设计理念
================================================================================

- 星链是工程流水线，不是创新研究
- 避免踩坑比创新更重要
- 历史最佳实践应该是制定规则的基础
- 宪法应该基于证据，而非主观判断
- 证据驱动规则 + 动态模型分配 = 质量与成本的最佳平衡
